## 💡 Key Takeaway
- 两类执行方式核心区别
  - 解析器调用执行（sh / bash）
    - 无需脚本拥有执行权限 `x`
    - 会忽略脚本首行 `#!/bin/bash` 释伴行，强制使用指定解析器运行
    - 支持当前目录、相对路径、绝对路径三种写法
  - 纯路径直接执行（./脚本 / 绝对路径/脚本）
    - 必须提前赋予文件执行权限 `chmod a+x`
    - 会读取脚本第一行 `#!/xxx` 声明，按指定解析器运行脚本
    - `.` 代表当前工作目录，`./` 表示当前目录下的文件
- 路径概念
  - 相对路径：以当前所在目录为基准，如 `hellowork.sh`、`./hellowork.sh`
  - 绝对路径：从根目录 `/` 完整定位文件，如 `/root/hellowork.sh`

```bash

##3.绝对路径

# bash解析器执行
##1.当前家目录
bash hellowork.sh

##2.相对路径
bash ./hellowork.sh

##3.绝对路径
bash /root/hellowork.sh 

# 仅路径执行
##1.提权
chmod a+x hellowork.sh

##2.相对路径
./hellowork.sh
 
# source 加载执行
##不会新建子shell，脚本变量直接作用于当前终端环境
source hellowork.sh #(等价于 . hellowrok.sh)
