## 💡 Key Takeaway
- 管道  |  作用：将前一条命令输出作为后一条命令输入，支持无限层嵌套
-  awk  自动以任意空白分割文本， $数字  代表第N列字段，解决cut多空格错乱问题
-  uniq -c  统计重复行数，必须先sort排序，仅能识别相邻重复内容
-  sort -n  数字升序， sort -nr  数字降序
- grep 基础正则不识别  + ，需加  -E  启用扩展正则
- 字段序号由日志格式决定，不可硬套模板

## ⚙️ Practice
```bash
##环境配置
cat > login.log <<EOF
user01 192.168.1.10 login 2026-06-28
user02 192.168.1.11 fail 2026-06-28
user01 192.168.1.10 fail 2026-06-28
user03 192.168.1.12 fail 2026-06-28
user02 192.168.1.11 fail 2026-06-28
user01 192.168.1.10 fail 2026-06-28
user04 192.168.1.13 login 2026-06-28
user02 192.168.1.11 login 2026-06-28
user01 192.168.1.10 login 2026-06-28
user05 192.168.1.14 fail 2026-06-28
EOF

##1.单层管道：过滤所有登录失败fail日志行
cat login.log | grep 'fail'

##2.双层管道：提取失败日志IP，数字升序排序
cat login.log | grep 'fail' | awk '{print $2}' | sort -n

##3.三层管道：统计每个IP失败次数，按次数降序展示
cat login.log | grep 'fail' | awk '{print $2}' | sort | uniq -c | sort -nr

##4.正则管道：提取日志内全部阿拉伯数字
cat login.log | grep -oE '[0-9]+'

##5.系统端口管道：查看监听端口，提取端口号、去重升序
ss -tulnp | grep -oE ':[0-9]+' | grep -E -o '[0-9]+' | sort -n | uniq

##6.用户统计管道：统计当前在线登录用户总人数
who | wc -l

##7.行截取嵌套：取前6行，筛选user01记录，仅输出用户名
head -n 6 login.log | grep 'user01' | awk '{print $1}'

##8.多层筛选：过滤失败登录≥2次的IP及对应次数
cat login.log | grep 'fail' | awk '{print $2}' | sort | uniq -c | awk '$1 >= 2'
 
🛠️ Troubleshooting
 
- 遇到的问题：
  - 执行 cat login.log | uniq  无法全局去重，重复内容依然存在
  - 执行 grep -o "[0-9]+" login.log  无任何数字输出
  - 日志存在连续空格时， cut -d' ' -f2  截取字段错乱
  - 模板命令  awk '{print $10}'  执行后无IP输出

- 报错原因：
  - uniq  只会消除相邻连续重复行，分散在文件各处的重复行无法识别
  - 默认基础grep不支持扩展正则元字符  + ，会把 + 当作普通文本匹配
  - cut将每一个单独空格视为分隔符，多个连续空格会造成字段序号偏移错位
  - 模板适配sshd标准日志，IP在第11列；本次自定义日志IP仅在第2列，字段序号不匹配

- 解决办法：
  - 先使用 sort 将相同内容排序聚拢，再管道交给uniq： cat login.log | sort | uniq
  - 增加 -E 参数启用扩展正则： grep -E -o "[0-9]+" login.log 
  - 改用awk自动识别任意空白分割，无需手动定义分隔符： cat login.log | grep fail | awk '{print $2}' 
  - 核对日志字段结构，修改awk输出列号为 $2
