🛠️ Troubleshooting
 
- 遇到的问题：
  - usermod: Permission denied
  - chown:invalid user: 'g+s'
  - 重定向报错：No such file or directory
  - cd /目录 无报错但重定向日志为空

- 报错原因：
  - 普通用户无权限修改用户组配置文件 /etc/passwd
  - g+s 是权限特殊标识，chown 仅修改属主属组，不能操作权限
  - 日志文件所存在目录不存在，系统无法创建目录
  - 目录存在，无标准错误输出，日志自然空白

- 解决办法：
  - 切换root用户执行，或命令前加sudo提权
  - 改用chown g+s /work 设置 SGID
  - 提前确定目录存在，不存在用mkdir -p 创建
  - 先填错不存在路径，才能捕获错误写入日志
